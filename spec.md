# 夹心层时间困境 — MVP Spec

## 它做什么

一个 3–5 分钟的单页网页小游戏：玩家在无尽弹出的任务清单里不停「清任务」，同时反复尝试点击永远排最后的「独处 10 分钟」——但每次都会被系统打断或改优先级，从而亲身体验「清单做不完、做回自己永远轮不到」的结构性时间困境。

## 给谁用

**同龄人试玩者**（同学、老师、项目答辩观众）——未必是夹心层本人，但通过短体验理解夹心层家庭「上有老下有小、清单驱动、无法休息」的压力，并在结尾留下反馈供论文与迭代使用。

## 核心流程

1. **打开 → 开始**：一句设定——「你是这一家的安排者。把今天过完。」
2. **清任务**：点击完成上班、做饭、辅导作业等任务；清单刚清完立刻弹出新任务；疲惫条下降，孩子落寞条缓慢上升。
3. **点「独处 10 分钟」**：进度条开始走 → 被紧急任务（加班 / 陪诊）打断 → 提示任务已插入清单前列。
4. **再清几轮 → 再点休息**：系统提示「已为您重新安排优先级」；休息按钮再次变灰、排至清单末尾。
5. **结尾屏**：不显示赢/输；展示行为统计（完成任务数、尝试休息次数、被打断次数）+ 一句收束文案 + 跳转或嵌入 2–3 道反馈题。

## 不在 MVP 里

- 拖拽操控布偶爸妈（点击完成任务即可）
- 工作 / 家庭 / 休息三套独立场景画面（用标签或图标区分）
- 爸妈双角色、祖辈完整任务线
- 三条状态条（MVP 只保留：疲惫条 + 孩子落寞条）
- 疲惫导致操作变慢、侧枝动画、音效、色调渐变
- 7 Beat 完整体验弧线、上帝视角切换各 stakeholder
- 访谈原话动态嵌入、调研数据驱动任务密度
- 隐藏小任务（如「喝咖啡」）、通关 / 最优解结局

## 怎么收集反馈

**结尾屏三题**（链到 Google 表单 / 问卷星，或页内 textarea）：

1. 玩的时候，你最想点但点不了的是什么？
2. 你有没有觉得「清单永远做不完」？（1–5 分）
3. 如果这是你家里的一天，你觉得谁最难受？为什么？

**游戏内行为统计**（`localStorage`，结尾展示）：完成任务数、尝试「独处」次数、被打断次数。

**线下试玩 5–10 人**：观察是否在中期主动寻找休息按钮、打断时是否愣住；简单记录表写入论文「用户测试」一节。
